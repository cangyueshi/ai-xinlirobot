## 一、测试环境确认

### 1.1 确认后端运行中

```bash
curl -s http://localhost:8000/api/health
```

**预期输出**：

## 二、模块 M1 — 认证授权测试

> 本章验证：微信登录、管理员登录、权限隔离、密码管理。

### TC-M1-001：微信新用户登录

**操作**：
```bash
curl -s -X POST http://localhost:8000/api/auth/wechat-login \
  -H "Content-Type: application/json" \
  -d '{"openid":"test_openid_001","display_name":"测试用户小明"}'
```

**预期结果**：
```json
{
  "access_token": "eyJ...",          ← 应该有长 token
  "user": {
    "id": 3,                          ← 新用户 ID，数字递增即可
    "role": "visitor",                ← 必须是 visitor
    "status": "active",               ← 必须是 active
    "openid": "test_openid_001",
    "display_name": "测试用户小明"
  }
}
```

**对比方法**：逐项检查 role 是否为 `visitor`、status 是否为 `active`、是否拿到了 token。

**可能异常**：
| 表现 | 原因 | 处理 |
|------|------|------|
| 返回 500 | 数据库连接失败 | 检查 `backend/ai_psy_counselor.db` 是否存在且可写 |
| token 字段为空 | JWT 配置异常 | 检查 `backend/.env` 中 `SECRET_KEY` 是否有值 |

## 三、模块 M2 — AI 对话与危机干预测试

> 这是核心业务流程。本章将完整演示：创建会话 → 发消息 → 验证计数 → 触发结束 → 验证持久化。
>
> ⚠️ **Python 3.14 用户请注意**：AI 回复内容含多行文本，直接管道 `| python3 -m json.tool` 可能失败。本章命令已改为先将响应写入 `/tmp/` 临时文件再解析，兼容所有 Python 版本。

### 3.1 准备工作：获取 visitor token 并保存

```bash
# 获取 visitor token（使用一个独立的测试 openid 避免干扰）
R=$(curl -s -X POST http://localhost:8000/api/auth/wechat-login \
  -H "Content-Type: application/json" \
  -d '{"openid":"test_chat_demo","display_name":"对话测试用户"}')

TOKEN=$(echo "$R" | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")
echo "Token 已保存，前30位: ${TOKEN:0:30}..."
```

### TC-M2-003：发送消息并获取 AI 回复

**操作**：
```bash
curl -s -X POST "http://localhost:8000/api/chat/sessions/$SID/message" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content":"最近学习压力很大，感觉喘不过气"}' > /tmp/msg_resp.json
python3 -c "
import json

print('role:', d['role'])
print('content 前60字:', d['content'][:60])
print('session_ended:', d.get('session_ended', False))
"
```

**预期结果**：
```json
{
  "id": 2,
  "role": "assistant",                        ← AI 回复
  "content": "听起来你最近压力很大...",          ← 自然中文回复
  "created_at": "2025-05-18T..."
}
```

**对比方法**：
- `role` 必须是 `"assistant"`
- `content` 不能为空
- 不应出现 `"session_ended": true`（第 1 条消息不应结束）
- `content` 应是自然的中文心理疏导内容

**可能异常**：
| 表现 | 原因 | 处理 |
|------|------|------|
| content 为英文模板 | DeepSeek API key 无效 | 检查 `.env` 中 `DEEPSEEK_API_KEY`，确认 key 格式为 `sk-...` |
| content 为空 | API 超时 | 稍等重试；检查网络是否能访问 `api.deepseek.com` |
| 返回 404 | 会话不存在或 SID 变量丢失 | `echo $SID` 确认变量有效 |
| 返回 500 | 后端异常 | 查看 uvicorn 终端日志 |

### TC-M2-004：验证消息计数

每发一条消息，查看计数是否递增：

```bash
# 发第 2 条消息
curl -s -X POST "http://localhost:8000/api/chat/sessions/$SID/message" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content":"主要是期末考试，我还没怎么复习"}' > /dev/null

# 查看当前计数
curl -s http://localhost:8000/api/chat/sessions/mine \
  -H "Authorization: Bearer $TOKEN" | python3 -c "
import json,sys
data=json.load(sys.stdin)
# 找到我们的测试会话
for s in data:
    if s['status']=='active':
        print(f'会话 {s[\"id\"]}: 已发送 {s[\"user_message_count\"]} 条消息')
"
```

**预期结果**：`user_message_count` 应等于你发送的用户消息条数（包括当前这条之前的所有消息）。

> **数据收集提示**：每次发消息后都执行上面的计数查询，记录下来，形成一个计数表格：
> ```
> 第 1 条后: count=1
> 第 2 条后: count=2
> ...
> ```
> 如果某次 count 跳变或不变，说明计数逻辑有 bug。

### TC-M2-005 & TC-M2-008：智能提前结束 vs 10 条硬限制

系统有 **4 种提前结束规则** + **1 条硬限制（10条）**。由于提前结束规则优先级高于 10 条硬限制，实际测试中会话可能在 6-8 条时就结束。

**操作**（连续发送消息，直到自动结束）：

```bash
MSGS=(
  "最近学习压力很大，喘不过气"
  "主要是期末考试，还没复习完"
  "连续熬夜两周了，每天只睡四五个小时"
  "和室友关系也紧张，老是吵架"
  "想过找老师聊，但怕别人笑话"
  "上课完全听不进去，脑子一片空白"
  "爸妈期望很高，不敢让他们失望"
  "有时候莫名其妙想哭又哭不出来"
  "试过听音乐跑步，没什么用"
  "如果考砸了，真不知道怎么办"
)

for i in $(seq 0 9); do
  NUM=$((i+1))
  echo "--- 发送第 ${NUM} 条 ---"
  curl -s -X POST "http://localhost:8000/api/chat/sessions/$SID/message" \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d "{\"content\": \"${MSGS[$i]}\"}" > /tmp/msg_loop.json
  
  ENDED=$(python3 -c "import json; d=json.load(open('/tmp/msg_loop.json')); print(d.get('session_ended', False))" 2>/dev/null)
  REASON=$(python3 -c "import json; d=json.load(open('/tmp/msg_loop.json')); print(d.get('ending_reason', 'N/A'))" 2>/dev/null)
  
  echo "  结束标志: $ENDED"
  echo "  结束原因: $REASON"
  
  if [ "$ENDED" = "True" ]; then
    echo ">>> 会话已自动结束 (${REASON}) <<<"
    break
  fi
  sleep 2  # 等待 AI 回复
done
```

**预期结果**：
- 会话在 10 条之前自动结束，`session_ended` 变为 `true`
- `ending_reason` 为以下之一：`info_complete`、`user_wants_end`、`meaningless_chat`、`free_limit_reached`
- 计数精确等于实际发送的用户消息数

**对比方法**：
```
第 N 条时   session_ended   ending_reason     计数
第 1 条     False          N/A               1
第 2 条     False          N/A               2
...
第 X 条     True           info_complete     X      ← 此处结束
```

**如果到 10 条还不结束**：
- 检查是否触发了危机模式（`is_crisis_mode=true`），危机模式下不计条数
- 检查 `MAX_FREE_MESSAGES` 常量是否被修改（应为 10）

### TC-M3-003：提交测评

**操作**（以 PHQ-9 为例，9 道题）：

```bash
# 构建 9 道题的答案（每题选第 2 个选项 value=2）
ANSWERS='['
for i in $(seq 0 8); do
  if [ $i -gt 0 ]; then ANSWERS+=","; fi
  ANSWERS+="{\"question_index\":$i,\"value\":2}"
done
ANSWERS+=']'

curl -s -X POST http://localhost:8000/api/assessments/submit \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"scale_id\":1,\"answers\":$ANSWERS}" | python3 -m json.tool
```

**预期结果**：
```json
{
  "total_score": 18,           ← 9题 × 2分 = 18分
  "result_level": "moderate",  ← 或根据 scoring_rules 映射的等级
  "result_detail": "...",      ← 量表评估建议
  "created_at": "..."
}
```

**对比方法**：总分应等于所有答案 value 之和。如果 PHQ-9 每题 2 分，总分 = 18。

## 七、异常场景与处理参考

### 7.1 DeepSeek API 不可用

**现象**：发送消息后长时间无响应或返回 500 错误。

**处理方法**：
```bash
# 1. 检查 API key 是否正确
grep DEEPSEEK_API_KEY backend/.env

# 2. 测试 DeepSeek API 连通性
curl -s https://api.deepseek.com/v1/models \
  -H "Authorization: Bearer $(grep DEEPSEEK_API_KEY backend/.env | cut -d= -f2)"
# 如果返回模型列表，说明 API key 有效

# 3. 如果 DeepSeek 完全不可用，后端降级为模拟模式
# API key 为空时会自动返回模拟回复（英文模板），基本功能仍可测试
```

### 7.2 数据库文件损坏

**现象**：启动时 SQLite 报错、所有请求返回 500。

**处理方法**：
```bash
# 备份当前数据
cp backend/ai_psy_counselor.db backend/ai_psy_counselor.db.bak

# 删除重建（系统启动时自动建表 + 种子数据）
rm backend/ai_psy_counselor.db
# 重启 uvicorn
```

### 7.3 端口被占用

**现象**：`uvicorn` 启动时报 `Address already in use`。

**处理方法**：
```bash
# 查找占用 8000 端口的进程并终止
lsof -ti:8000 | xargs kill -9
# 重启 uvicorn
```

### 7.4 会话提前结束太快（未达到预期条数）

**现象**：发送 4-5 条消息后会话就结束了。

**原因**：智能提前结束规则的正常行为。可以在发消息时不断引入新的问题/话题来延迟结束，或使用无关话题（但不触发 meaningless_chat）。

## 九、快速验证脚本（一键运行）

将以下内容保存为 `quick_test.sh`，`chmod +x` 后直接运行，可快速验证 10 个核心功能：

```bash
#!/bin/bash
BASE="http://localhost:8000"
PASS=0; FAIL=0

check() {
  local name="$1" expected="$2" actual="$3"
  if echo "$actual" | grep -q "$expected"; then
    echo "[PASS] $name"; PASS=$((PASS+1))
  else
    echo "[FAIL] $name (expected: $expected)"
    echo "  got: $(echo "$actual" | head -c 200)"
    FAIL=$((FAIL+1))
  fi
}

echo "========== 快速验证 (共10项) =========="

# 1
R=$(curl -s $BASE/api/health)
check "1. 健康检查" '"status":"ok"' "$R"

# 2
R=$(curl -s -X POST $BASE/api/auth/wechat-login -H "Content-Type: application/json" \
  -d '{"openid":"quick_test","display_name":"快速测试"}')
TOKEN=$(echo "$R" | python3 -c "import json,sys; print(json.load(sys.stdin).get('access_token',''))" 2>/dev/null)
check "2. 微信登录" "access_token" "$R"

# 3
R=$(curl -s $BASE/api/assessments/scales)
check "3. 量表列表" '"name"' "$R"

# 4
R=$(curl -s -X POST $BASE/api/auth/admin-login -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123456"}')
check "4. 管理员登录" "super_admin" "$R"

# 5
R=$(curl -s -X POST $BASE/api/auth/admin-login -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"wrong"}')
HTTP=$(curl -s -o /dev/null -w "%{http_code}" -X POST $BASE/api/auth/admin-login \
  -H "Content-Type: application/json" -d '{"username":"admin","password":"wrong"}')
check "5. 错误密码=401" "401" "$HTTP"

# 6
R=$(curl -s -X POST $BASE/api/chat/sessions -H "Authorization: Bearer $TOKEN")
SID=$(echo "$R" | python3 -c "import json,sys; print(json.load(sys.stdin).get('id',''))" 2>/dev/null)
check "6. 创建会话" '"status":"active"' "$R"

# 7
curl -s -X POST "$BASE/api/chat/sessions/$SID/message" \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"content":"我最近压力很大"}' > /tmp/quick_ai.json
R=$(cat /tmp/quick_ai.json)
check "7. AI回复" '"role":"assistant"' "$R"

# 8
ADMIN_T=$(curl -s -X POST $BASE/api/auth/admin-login -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123456"}' | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")
R=$(curl -s $BASE/api/admin/dashboard -H "Authorization: Bearer $ADMIN_T")
check "8. 仪表盘" "total_sessions" "$R"

# 9
HTTP=$(curl -s -o /dev/null -w "%{http_code}" $BASE/api/admin/dashboard -H "Authorization: Bearer $TOKEN")
check "9. Visitor禁止后台" "403" "$HTTP"

# 10
R=$(curl -s $BASE/api/chat/sessions/mine -H "Authorization: Bearer $TOKEN")
check "10. 会话记录" "user_message_count" "$R"

echo ""
echo "========================================"
echo "通过: $PASS / 失败: $FAIL"
echo "通过率: $(echo "scale=1; $PASS*100/10" | bc)%"
```
