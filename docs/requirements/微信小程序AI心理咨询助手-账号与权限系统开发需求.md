# 微信小程序 AI 心理咨询助手 — 账号与权限系统开发需求

**版本**：V1.0  
**生效日期**：2026 年 5 月 17 日  
**适用范围**：微信小程序 AI 心理咨询助手（前端）+ 管理后台（网页端）

---

## 一、登录体系与端口隔离

### 1.1 小程序前端（用户端）

- **登录入口**：仅保留**微信一键授权登录**一个入口
- **账号创建**：首次登录自动创建用户账号，即时可用，无需审核
- **获取信息**：微信昵称、微信头像
- **不收集**：手机号、邮箱等额外注册信息

### 1.2 管理后台（网页端）

- **访问方式**：独立域名 / 路径访问
- **登录方式**：仅支持**账号密码登录**
- **不开放**：微信扫码登录、第三方 OAuth 登录

### 1.3 端口隔离规则（严格禁止）

| 规则 | 说明 |
|------|------|
| 用户账号 → 后台 | ❌ 禁止登录管理后台 |
| 咨询师账号 → 小程序 | ❌ 禁止登录小程序前端 |
| 管理员账号 → 小程序 | ❌ 禁止登录小程序前端 |

> **原则**：三个端口 —— 用户端（小程序）、咨询师工作端（后台）、管理端（后台）—— 互相隔离，不允许任何跨端口登录行为。

## 二、账号创建规则

### 2.1 普通用户账号

| 项目 | 说明 |
|------|------|
| **创建方式** | 用户在微信小程序中点击「微信授权登录」后**自动创建** |
| **审核** | 无需审核，即时可用 |
| **激活** | 创建即激活，无需额外操作 |
| **限制** | 无注册门槛，无数量限制 |

### 2.2 咨询师账号

#### 2.2.1 创建方式

- **不开放**任何自主注册入口
- **全部由超级管理员在后台手动创建**

#### 2.2.2 创建时必填字段

| 字段 | 说明 |
|------|------|
| 咨询师姓名 | 真实姓名或执业用名 |
| 登录账号 | 用于登录管理后台的唯一标识 |
| 初始密码 | 由管理员设定，创建完成后将账号密码告知对应咨询师 |
| 简介 | 简短的个人介绍，展示给用户 |
| 擅长领域 | 如：亲子关系、焦虑情绪、青少年心理等 |

#### 2.2.3 创建后状态

- 创建完成后账号**立即生效**
- 由**管理员负责**将账号密码告知对应咨询师（线下或内部通讯工具）
- 支持后台修改咨询师信息、重置密码

#### 2.2.4 账号管理操作

| 操作 | 说明 |
|------|------|
| 修改信息 | 修改姓名、简介、擅长领域等基本信息 |
| 重置密码 | 管理员直接重置，无需旧密码验证 |
| 临时禁用 | 账号立即无法登录，保留所有历史数据 |
| 永久删除 | 彻底移除账号，原有数据保留不删除 |

### 2.3 超级管理员账号

#### 2.3.1 创建方式

- **系统部署时预先内置**唯一初始账号
- 开发完成后**直接交付**初始账号密码给项目负责人

#### 2.3.2 首次登录

- 负责人首次登录后台后，**强制要求修改密码**
- 未修改密码前，无法执行任何管理操作

#### 2.3.3 权限与限制

| 项目 | 说明 |
|------|------|
| 系统最高权限 | 拥有后台全部功能的操作权限 |
| 不可修改 | 该账号的基本信息无法被其他账号修改 |
| 不可删除 | 该账号无法被其他账号删除 |
| 自创建次级管理员 | 超级管理员可在后台手动创建**次级管理员账号** |

#### 2.3.4 次级管理员

- 由超级管理员手动创建
- 权限由超级管理员**按模块分配**（如：仅可管理用户数据、仅可查看统计报表等）
- 支持随时收回权限或删除账号

## 三、权限隔离规则（严格执行）

### 3.1 角色权限矩阵

| 角色 | 可查看数据范围 | 可操作权限 |
|------|---------------|-----------|
| **普通用户** | 仅自己的聊天记录、测评结果、订单信息 | 使用小程序功能、修改个人信息 |
| **咨询师** | 仅自己接待过的来访者的聊天记录、咨询笔记、订单 | 接单、回复咨询、填写咨询笔记、查看自己的营收统计 |
| **超级管理员** | 全站所有数据（所有用户、所有咨询师的全部记录） | 管理所有账号、查看所有数据、导出数据、修改系统设置、管理订单与内容 |
| **次级管理员** | 按所分配的模块范围 | 按所分配的操作权限 |

### 3.2 隔离规则要点

- **用户与用户之间隔离**：用户 A 无法查看用户 B 的任何数据
- **咨询师之间隔离**：咨询师 A 无法查看咨询师 B 的来访者数据
- **咨询师不可越级**：咨询师不可查看非自己接待的用户的数据
- **管理员全量可见**：超级管理员可查看所有数据（受合规要求约束）
- **数据按层过滤**：任何 SQL / API 查询都必须附带当前用户的角色与权限过滤条件，禁止在后端做一次全量查询后再在前端过滤

## 四、补充必做功能

### 4.1 数据实时同步与检索

- 所有聊天记录、咨询记录、用户数据**实时同步**至后台数据库
- 支持按以下条件筛选与检索：

| 筛选项 | 说明 |
|--------|------|
| 时间范围 | 按开始时间 / 结束时间筛选 |
| 用户维度 | 按用户昵称、微信 openid 检索 |
| 咨询师维度 | 按咨询师姓名、ID 检索 |
| 关键词 | 按聊天内容关键词搜索（仅管理员可用） |

### 4.2 数据导出（Excel 格式）

- 超级管理员支持**一键导出**任意筛选范围内的数据
- 导出字段包括但不限于：
  - 用户信息（昵称、openid、注册时间）
  - 咨询师信息（姓名、擅长领域、接单数量）
  - 对话记录（时间、内容摘要、情绪标签）
  - 订单记录（金额、状态、时间）
- 导出操作记录日志，留存操作人、时间、导出范围

### 4.3 账号禁用与数据保留

| 操作 | 账号状态 | 数据状态 |
|------|---------|---------|
| 禁用账号 | 立即不可登录，无法使用任何功能 | 原有数据保留，不删除 |
| 删除账号 | 账号被移除 | 原有数据保留，标记为「已删除用户」 |
| 重新启用 | 恢复正常使用 | 数据完整恢复 |

### 4.4 密码找回功能

**适用范围**：仅限管理员 / 咨询师账号（普通用户使用微信登录，无密码）

**找回流程**：

1. 在登录页面点击「忘记密码」
2. 输入预留的邮箱地址
3. 系统向该邮箱发送重置链接
4. 点击链接进入重置页面，设置新密码
5. 密码重置成功后，所有现有登录会话立即失效

**技术约束**：

- 重置链接有效期：30 分钟
- 验证码发送频率限制：每 60 秒 1 次
- 单日最多发送 5 次重置邮件

### 4.5 安全补充

- 所有密码以哈希存储（推荐 bcrypt / Argon2），禁止明文存储
- API 端点需校验请求来源域名（Referer / Origin 校验），防止跨站请求
- 小程序端与后台端使用不同的 AppSecret / JWT Secret
- 登录 Token 有效期建议：小程序端 7 天，后台端 8 小时
