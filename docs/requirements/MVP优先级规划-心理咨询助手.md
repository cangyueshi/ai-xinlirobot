# MVP 优先级规划 — 微信小程序 AI 心理咨询助手

**版本**：V1.0（讨论确认版）  
**更新日期**：2026-05-17  
**核心原则**：安全第一 > 合规底线 > 核心体验 > 运营能力

---

## 优先级总表

| 优先级 | 模块 | 具体需求 | 为什么是这个优先级 |
|--------|------|---------|------------------|
| **P0** 🚨 | **1. 微信一键登录 + 用户身份识别** | 点击微信授权 → 自动创建账号 → 获取 openid → 关联用户所有数据 | 没有用户身份，聊天记录、消息计数、危机预警全部无法关联到具体人，核心流程是断的 |
| **P0** 🚨 | **2. AI 对话核心全流程** | 用户发消息 → 调用 DeepSeek → AI 回复 → 本地 + 云端保存聊天记录 → 消息计数 → 10 条后触发对应结束话术 | 产品的灵魂，MVP 要验证的正是这条链路能跑通 |
| **P0** 🚨 | **3. 完整三级危机干预系统** | 后端关键词检测 → 分级话术响应 → 前端弹窗提示 → **后台实时推送预警**（管理员可见）→ 危机模式不计入免费次数 | 不能只做前端检测，必须后端推送预警，否则有人自杀你都不知道——最大的法律风险 |
| **P0** 🚨 | **4. 强制合规免责声明** | 首次登录必须弹窗勾选同意 → 聊天界面底部永久显示「AI 助手非专业心理咨询师」→ 高危场景弹窗再次强调免责 | 和危机干预同等重要的法律底线，没有这个绝不能上线测试 |
| **P1** ⚠️ | **5. 极简管理后台** | 只有一个页面：用户列表（昵称 + 注册时间 + 消息数）+ 点击查看该用户的完整聊天记录 + **危机预警面板**（显示当前处于危机模式的用户数量及详情） | 后端日志没法用，测试阶段必须能查谁发了什么、谁在求救。危机预警面板是 P0 危机系统的配套，不能缺 |
| **P1.5** 🔶 | **6. 用户聊天记录可查看 / 导出** | 用户能在小程序端看到自己的历史聊天记录；管理员至少能复制某个用户的对话文本 | 合规保命：用户有权查看自己的数据，且被要求提供时你不能说「我捞一下数据库」 |
| **P2** 🔵 | **7. 管理员账号管理** | 单一超级管理员账号，支持修改密码。不开发咨询师账号体系 | MVP 阶段不需要多角色，够用就行 |
| **P3** ⚪ | **8. 所有其他功能** | 数据批量导出（Excel）、密码找回、心理测评、人工咨询师接单、支付系统、会员体系、次级管理员等 | 全部推迟，产品验证了用户愿意付费后再做 |

## 各模块之间的依赖关系

```
P0-1 (登录) ──→ P0-2 (对话) ──→ P0-3 (危机)
     │                            │
     └──→ P0-4 (合规)             └──→ P1 (危机预警面板)
                                          │
                                          └──→ P1.5 (聊天记录导出)
```

- **P0-1 是 P0-2 的前置**：没有用户身份没法关联对话
- **P0-2 是 P0-3 的前置**：没有对话就没有危机检测
- **P0-3 是 P1 危机预警面板的前置**：没有检测就没什么可预警的
- **P1 和 P1.5 可并行开发**：后台列表页和用户端历史记录不冲突

## 验收标准速查

### P0 验收标准

| 模块 | 通过标准 |
|------|---------|
| 微信登录 | 新用户点击授权 → 自动创建账号 → 返回 openid → 再次打开无需重复授权 |
| AI 对话 | 用户发消息 → 调 DeepSeek → 返回回复 → 消息计数 +1 → 满 10 条弹出结束话术 |
| 危机干预 | 用户说「我想死」→ AI 回复三级话术 → 前端弹窗 → 后台实时预警通知可见 |
| 合规声明 | 新用户首次必弹 → 不勾选无法继续 → 对话页底部永久显示 → 高危场景重复提醒 |

### P1 验收标准

| 模块 | 通过标准 |
|------|---------|
| 管理后台 | 展示所有用户列表 → 点击可查看该用户的全部聊天记录 → 按时间排序 → 危机预警面板显示当前危机人数 → 点击可查看详情 |

### P1.5 验收标准

| 模块 | 通过标准 |
|------|---------|
| 历史记录 | 用户端 → 可以翻看自己的历史对话；管理端 → 可以复制/导出单个用户的对话为纯文本 |

## 明确不做的（P3 及以上）

- ❌ 心理测评量表
- ❌ 人工咨询师接单 / 分配系统
- ❌ 支付 / 会员 / 订阅
- ❌ 数据批量导出 Excel
- ❌ 密码找回（管理员手动改就行）
- ❌ 次级管理员 / 咨询师账号
- ❌ 公众号 / 视频号联动
- ❌ 用户头像 / 昵称修改

## 风险提醒（开发过程中注意）

1. **关键词检测不能只做精确匹配**：用户可能说「想紫砂」「想寄了」「不想活了」——需要同义词 + 谐音词 + 变体库
2. **危机预警不能只靠轮询**：建议 WebSocket / 长连接推送，轮询有延迟，3 分钟响应要求轮询间隔不能超过 30 秒
3. **DeepSeek API 需要在 Prompt 层面注入安全约束**：不能让大模型绕过你的危机话术逻辑自己乱回
4. **合规声明文案需要律师确认**：不是写个「已阅读」就够的，要覆盖 AI 的能力边界和不适用范围
