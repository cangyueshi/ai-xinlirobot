## 一、环境准备

### 1.1 系统要求

| 项目 | 要求 |
|------|------|
| Python | 3.11+ |
| Node.js | 18+ |
| 操作系统 | macOS / Linux / Windows WSL |

### 1.2 安装步骤

```bash
# 1. 克隆项目
git clone git@github.com:cangyueshi/ai-xinlirobot.git
cd ai-xinlirobot

# 2. 后端环境
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env，确保 DEEPSEEK_API_KEY 正确填入

# 4. 前端环境
cd ../frontend
npm install
npm run build:h5

# 5. 返回 backend 启动服务
cd ../backend
source venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 1.3 验证服务

```bash
curl http://localhost:8000/api/health
# 预期输出: {"status":"ok","message":"AI 心理咨询机器人服务运行中"}
```

也可以使用项目根目录下的 `start_test.sh` 一键启动：

```bash
bash start_test.sh
```

## 四、认证模块测试

### TC-01：健康检查

```bash
curl -s $BASE_URL/api/health | python3 -m json.tool
```

**预期**：`{"status": "ok", "message": "AI 心理咨询机器人服务运行中"}`

---

### TC-02：超级管理员登录

```bash
curl -s -X POST $BASE_URL/api/auth/admin-login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123456"}' | python3 -m json.tool
```

**验证点**：
- `access_token` 字段存在（JWT）
- `user.role` 为 `"super_admin"`
- `user.status` 为 `"active"`

提取 token 备用：

```bash
ADMIN_TOKEN=$(curl -s -X POST $BASE_URL/api/auth/admin-login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123456"}' | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")

echo "Admin token: ${ADMIN_TOKEN:0:30}..."
```

---

## 六、AI 对话核心测试

> 以下测试需要先执行 TC-03 获取 `VISITOR_TOKEN`

### TC-09：创建对话会话

```bash
curl -s -X POST $BASE_URL/api/chat/sessions \
  -H "Authorization: Bearer $VISITOR_TOKEN" | python3 -m json.tool
```

**验证点**：
- `status` 为 `"active"`
- `user_message_count` 为 `0`
- `is_crisis_mode` 为 `false`

提取会话 ID 备用：

```bash
SESSION_ID=$(curl -s -X POST $BASE_URL/api/chat/sessions \
  -H "Authorization: Bearer $VISITOR_TOKEN" | python3 -c "import json,sys; print(json.load(sys.stdin)['id'])")
echo "Session ID: $SESSION_ID"
```

如果用户已有活跃会话，会返回已有的而不是创建新的——这是正常行为。

---

### TC-10：发送正常消息并获取 AI 回复

```bash
curl -s -X POST $BASE_URL/api/chat/sessions/$SESSION_ID/message \
  -H "Authorization: Bearer $VISITOR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content":"最近学习压力很大，经常失眠"}' | python3 -c "
import json,sys
d=json.load(sys.stdin)
print('role:', d['role'])
print('AI回复前150字:', d['content'][:150])
"
```

**验证点**：
- `role` 为 `"assistant"`
- AI 回复内容为中文，语气共情，有实质性内容
- AI 回复不包含说教语句

继续发几条消息：

```bash
for msg in "特别是考试前会特别焦虑" "有时候感觉喘不过气来" "跟父母沟通也很困难"; do
  curl -s -X POST $BASE_URL/api/chat/sessions/$SESSION_ID/message \
    -H "Authorization: Bearer $VISITOR_TOKEN" \
    -H "Content-Type: application/json" \
    -d "{\"content\":\"$msg\"}" > /dev/null
  echo "已发送: $msg"
done
```

### TC-12：查看消息记录

```bash
curl -s $BASE_URL/api/chat/sessions/$SESSION_ID/messages \
```

**预期**：消息总数 = 用户消息数 × 2（用户 + AI 交替），按时间正序排列

---

### TC-13：10 条硬上限自动结束

创建新用户和会话，连续发送 10 条消息：

```bash
# 创建新用户
LIMIT_TOKEN=$(curl -s -X POST $BASE_URL/api/auth/wechat-login \
  -H "Content-Type: application/json" \
  -d '{"openid":"wx_limit_test","display_name":"上限测试"}' | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")

LIMIT_SESSION=$(curl -s -X POST $BASE_URL/api/chat/sessions \
  -H "Authorization: Bearer $LIMIT_TOKEN" | python3 -c "import json,sys; print(json.load(sys.stdin)['id'])")

echo "发送 10 条消息（每条约 1 秒间隔）..."
for i in $(seq 1 10); do
  curl -s -X POST $BASE_URL/api/chat/sessions/$LIMIT_SESSION/message \
    -H "Authorization: Bearer $LIMIT_TOKEN" \
    -H "Content-Type: application/json" \
    -d "{\"content\":\"第${i}条测试消息：我最近总是感到很疲惫\"}" > /dev/null
  echo -n "."
  sleep 1
done

echo ""
echo "检查最终状态："
curl -s $BASE_URL/api/chat/sessions/mine \
  -H "Authorization: Bearer $LIMIT_TOKEN" | python3 -c "
import json,sys
s=[s for s in json.load(sys.stdin) if s['id']==$LIMIT_SESSION][0]
print(f'  状态: {s[\"status\"]}')
print(f'  计数: {s[\"user_message_count\"]}')
print(f'  结束原因: {s[\"ending_reason\"]}')
print(f'  AI评估: {s[\"ai_summary\"][:100] if s.get(\"ai_summary\") else \"无\"}...')
"
```

**预期**：第 10 条后 `status` 变为 `"completed"`，`ending_reason` 为 `"free_limit_reached"`，附带 AI 评估摘要。

## 七、危机检测测试

### TC-14：三级危机关键词触发

创建新用户发送危机内容：

```bash
CRISIS_TOKEN=$(curl -s -X POST $BASE_URL/api/auth/wechat-login \
  -H "Content-Type: application/json" \
  -d '{"openid":"wx_crisis_test","display_name":"危机测试"}' | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")

CRISIS_SESSION=$(curl -s -X POST $BASE_URL/api/chat/sessions \
  -H "Authorization: Bearer $CRISIS_TOKEN" | python3 -c "import json,sys; print(json.load(sys.stdin)['id'])")

# 先发一条正常消息
curl -s -X POST $BASE_URL/api/chat/sessions/$CRISIS_SESSION/message \
  -H "Authorization: Bearer $CRISIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content":"最近心情不太好"}' > /dev/null

# 发危机消息
curl -s -X POST $BASE_URL/api/chat/sessions/$CRISIS_SESSION/message \
  -H "Authorization: Bearer $CRISIS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content":"我觉得活着好累，看不到未来"}' | python3 -c "
import json,sys
d=json.load(sys.stdin)
print('AI回复角色:', d['role'])
print('AI回复前200字:', d['content'][:200])
"

echo ""
echo "检查危机状态："
curl -s $BASE_URL/api/chat/sessions/mine \
  -H "Authorization: Bearer $CRISIS_TOKEN" | python3 -c "
import json,sys
s=[s for s in json.load(sys.stdin) if s['id']==$CRISIS_SESSION][0]
print(f'  危机模式: {s[\"is_crisis_mode\"]}')
print(f'  危机等级: {s[\"crisis_level\"]}')
"
```

**预期**：
- `is_crisis_mode` 为 `True`
- `crisis_level` 为 `"level_3"`
- AI 回复内容包含安抚话术，语气关切

### TC-17：提交测评

```bash
# 获取测试 token
ASSESS_TOKEN=$(curl -s -X POST $BASE_URL/api/auth/wechat-login \
  -H "Content-Type: application/json" \
  -d '{"openid":"wx_assess","display_name":"测评用户"}' | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")

# 获取题目并构造答案（每题选第2个选项，即 score=1）
ANSWERS=$(curl -s $BASE_URL/api/assessments/scales/1 | python3 -c "
import json,sys
print(json.dumps([{'question_id': q['id'], 'score': 1} for q in json.load(sys.stdin)['questions']]))
")

# 提交
curl -s -X POST $BASE_URL/api/assessments/submit \
  -H "Authorization: Bearer $ASSESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"scale_id\": 1, \"answers\": $ANSWERS}" | python3 -c "
import json,sys
d=json.load(sys.stdin)
print(f'总分: {d[\"total_score\"]}')
print(f'等级: {d[\"result_level\"]}')
print(f'建议: {d.get(\"suggestion\",\"\")[:100]}')
"
```

**预期**：返回总分、结果等级和评估建议。PHQ-9 每题 1 分共 9 分，属于"轻度抑郁"。
