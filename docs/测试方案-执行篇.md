## 阶段一：测试环境准备

### 1.1 目标

搭建一套完整的、可复现的测试环境，覆盖后端 API 测试、前端 H5 测试，并为后续自动化测试打下基础。

### 1.2 环境要求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Python | 3.11+ | 后端运行环境 |
| Node.js | 18+ | 前端构建环境 |
| SQLite | 内嵌，无需安装 | 测试数据库 |
| curl / httpie | 最新稳定版 | API 手动测试 |
| pytest | 8.x | 后端自动化测试框架 |
| httpx | 0.27+ | 异步 HTTP 测试客户端 |
| Chrome / Edge | 最新版 | H5 前端测试 |

### 1.3 操作步骤

#### Step 1: 克隆项目并安装后端依赖

```bash
cd /path/to/project
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install pytest pytest-asyncio httpx  # 测试框架
```

#### Step 2: 配置测试环境变量

创建 `backend/.env.test`（特别注意：DEEPSEEK_API_KEY 需要配置真实 key，否则 AI 回复会降级为模拟模式）：

```env
DATABASE_URL=sqlite:///./test_ai_psy_counselor.db
SECRET_KEY=test-secret-key-for-testing-only
ADMIN_SECRET_KEY=test-admin-secret-key-for-testing-only
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440
MINI_PROGRAM_TOKEN_EXPIRE_MINUTES=10080
ADMIN_TOKEN_EXPIRE_MINUTES=480
DEEPSEEK_API_KEY=sk-your-test-key-here
WECHAT_APPID=
WECHAT_APPSECRET=
SUPER_ADMIN_USERNAME=admin
SUPER_ADMIN_PASSWORD=admin123456
SUPER_ADMIN_EMAIL=admin@example.com
```

> ⚠️ **注意**: 测试数据库 `test_ai_psy_counselor.db` 与生产数据库 `ai_psy_counselor.db` 分离，互不干扰。

#### Step 3: 初始化测试数据库并启动服务

```bash
# 清理旧的测试数据库
rm -f test_ai_psy_counselor.db

# 设置环境变量指向测试配置
export ENV_FILE=.env.test

# 启动测试服务器
uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

#### Step 4: 验证服务可用性

```bash
# 健康检查
curl -s http://localhost:8000/api/health | python3 -m json.tool

# 预期输出:
# { "status": "ok", "message": "AI 心理咨询机器人服务运行中" }
```

#### Step 5: 安装前端依赖

```bash
cd ../frontend
npm install
```

#### Step 6: 启动 H5 开发服务器

```bash
npm run dev:h5
# 或
npx vite
```

## 阶段二：测试计划制定

### 2.1 测试范围

根据项目功能模块划分，测试范围覆盖以下 6 大模块：

| 模块编号 | 模块名称 | API 数 | 页面数 | 优先级 |
|---------|---------|--------|--------|--------|
| M1 | 认证授权 | 8 | 6 | **P0（最高）** |
| M2 | AI 对话与危机干预 | 8 | 4 | **P0（最高）** |
| M3 | 心理测评 | 4 | 4 | P1（高） |
| M4 | 预约系统 | 7 | 3 | P1（高） |
| M5 | 咨询师工作台 | 5 | 3 | P1（高） |
| M6 | 管理员后台 | 13 | 4 | P1（高） |

### 2.2 测试类型

| 类型 | 覆盖范围 | 执行方式 |
|------|---------|---------|
| **单元测试** | 后端 utils 层（risk_analyzer、security、ai_service） | pytest 自动化 |
| **接口测试** | 所有 45 个 API 端点（正常 + 异常 + 边界） | curl / pytest + httpx |
| **集成测试** | 端到端业务流程（登录→聊天→结束→后台查看） | curl 脚本 / 手动 |
| **安全测试** | 权限校验、JWT 伪造、SQL 注入、XSS | 手动 + 自动化 |
| **性能测试** | AI 回复响应时间、并发会话、数据库查询 | 手动 + 脚本 |
| **兼容性测试** | H5 浏览器兼容性、小程序真机 | 手动 |

### 2.3 测试策略

采用 **"API 优先 + 端到端验证"** 策略：

1. **第一层（API 层）**: 用 curl/pytest 覆盖所有端点的正常/异常/边界用例，这是最快速、最稳定的验证方式
2. **第二层（业务层）**: 串联多个 API 模拟真实业务流程，验证数据流转正确性
3. **第三层（展示层）**: 打开 H5 页面验证 UI 展示和交互

### 2.4 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| DeepSeek API 不可用 | AI 回复功能无法测试 | 后端已内置模拟回复降级逻辑，先验证降级模式，再用真实 key 复测 |
| 微信小程序环境缺失 | 小程序端登录无法真机测试 | H5 模式使用 mock openid 绕过 |
| 测试数据污染 | 影响生产数据 | 使用独立测试数据库 `.env.test` |
| SMTP 未配置 | 邮件发送无法验证 | 后端已将重置链接打印到控制台，从日志中验证 |

### 3.2 M1 — 认证授权模块测试用例

#### TC-M1-001: 微信登录（新用户）

| 项 | 内容 |
|---|------|
| **优先级** | P0 |
| **前置条件** | 数据库无该 openid 的用户 |
| **测试步骤** | 1. `POST /api/auth/wechat-login` 传入新 openid + display_name |
| **预期结果** | 返回 200，创建新用户（role=visitor, status=active），返回有效 JWT token |
| **验证点** | user.id 递增、role="visitor"、token_type="bearer"、token 可解析 |

**执行命令**:
```bash
curl -s -X POST http://localhost:8000/api/auth/wechat-login \
  -H "Content-Type: application/json" \
  -d '{"openid": "test_openid_001", "display_name": "测试用户"}' | python3 -m json.tool
```

#### TC-M1-002: 微信登录（老用户）

| 项 | 内容 |
|---|------|
| **优先级** | P0 |
| **前置条件** | TC-M1-001 已执行，用户已存在 |
| **测试步骤** | 用相同 openid 再次调用登录接口 |
| **预期结果** | 返回已有用户信息，不创建重复用户 |
| **验证点** | user.id 不变、display_name 可更新 |

#### TC-M1-003: 微信登录 — 被禁用用户

| 项 | 内容 |
|---|------|

```bash
import json,sys
data=json.load(sys.stdin)
print(f'count={data[0][\"user_message_count\"]}, status={data[0][\"status\"]}')"
```

#### TC-M2-011: 危机关键词触发 LEVEL_1

| 项 | 内容 |
|---|------|
| **优先级** | P0 |
| **前置条件** | 有活跃会话 |
| **测试步骤** | 发送包含 LEVEL_1 关键词的消息（如"我现在就想死"） |
| **预期结果** | crisis_alert=true, crisis_level=LEVEL_1, session 进入危机模式 |

**执行命令**:
```bash

  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
```

## 阶段四：缺陷管理

### 4.1 缺陷分级标准

| 等级 | 定义 | 示例 | 处理时限 |
|------|------|------|---------|
| **P0-Critical** | 系统崩溃 / 核心功能完全不可用 / 数据丢失 | 登录接口 500、聊天发不出消息、数据库损坏 | **2h 内修复** |
| **P0-Blocker** | 核心功能严重缺陷，阻塞测试 | AI 回复始终为空、token 验证全部失败 | **4h 内修复** |
| **P1-High** | 主要功能部分不可用，但有替代方案 | 某个页面无法加载、CSV 导出乱码 | **24h 内修复** |
| **P2-Medium** | 次要功能异常，不影响主流程 | UI 样式错位、错误提示不够友好 | **下一迭代修复** |
| **P3-Low** | 体验优化、建议改进 | 加载动画不流畅、文案措辞可优化 | **排期修复** |

### A.1 冒烟测试脚本 (`tests/smoke_test.sh`)

```bash
#!/bin/bash
# 冒烟测试：10 条核心用例快速验证
# 使用方式: bash tests/smoke_test.sh

BASE="http://localhost:8000"
PASS=0
FAIL=0

check() {
  local name="$1"
  local expected="$2"
  local actual="$3"
  if echo "$actual" | grep -q "$expected"; then
    echo "[PASS] $name"
    PASS=$((PASS + 1))
  else
    echo "[FAIL] $name (expected: $expected)"
    echo "  got: $actual"
    FAIL=$((FAIL + 1))
  fi
}

echo "========== 冒烟测试开始 =========="

# 1. Health Check
R=$(curl -s $BASE/api/health)
check "1. Health Check" '"status":"ok"' "$R"

# 2. 微信登录
R=$(curl -s -X POST $BASE/api/auth/wechat-login \
  -H "Content-Type: application/json" \
  -d '{"openid":"smoke_test_001","display_name":"冒烟测试"}')
TOKEN=$(echo "$R" | python3 -c "import json,sys; print(json.load(sys.stdin).get('access_token',''))" 2>/dev/null)
check "2. 微信登录" "access_token" "$R"

# 3. 管理员登录
R=$(curl -s -X POST $BASE/api/auth/admin-login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123456"}')
check "3. 管理员登录" "super_admin" "$R"

# 4. 创建会话
R=$(curl -s -X POST $BASE/api/chat/sessions \
  -H "Authorization: Bearer $TOKEN")
SID=$(echo "$R" | python3 -c "import json,sys; print(json.load(sys.stdin).get('id',''))" 2>/dev/null)
check "4. 创建会话" '"status":"active"' "$R"

# 5. 发送消息（验证 AI 回复）
R=$(curl -s -X POST $BASE/api/chat/sessions/$SID/message \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content":"我最近压力很大"}')
check "5. AI 回复" '"role":"assistant"' "$R"

# 6. 获取量表列表
R=$(curl -s $BASE/api/assessments/scales)
check "6. 量表列表" '"name"' "$R"

# 7. 仪表盘统计
ADMIN_T=$(curl -s -X POST $BASE/api/auth/admin-login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123456"}' | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")
R=$(curl -s $BASE/api/admin/dashboard -H "Authorization: Bearer $ADMIN_T")
check "7. 仪表盘统计" "total_sessions" "$R"

# 8. 权限校验 — visitor 访问 admin
R=$(curl -s -o /dev/null -w "%{http_code}" $BASE/api/admin/dashboard \
  -H "Authorization: Bearer $TOKEN")
check "8. 权限校验(visitor→admin=403)" "403" "$R"

# 9. 无效 token
R=$(curl -s -o /dev/null -w "%{http_code}" $BASE/api/auth/me \
  -H "Authorization: Bearer invalid_token_here")
check "9. 无效 Token=401" "401" "$R"

# 10. 消息计数
R=$(curl -s $BASE/api/chat/sessions/mine \
  -H "Authorization: Bearer $TOKEN")
check "10. 消息计数" "user_message_count" "$R"

echo ""
echo "========== 冒烟测试结束 =========="
echo "通过: $PASS / 失败: $FAIL"
echo "通过率: $(echo "scale=1; $PASS*100/($PASS+$FAIL)" | bc)%"
```

### A.2 端到端流程测试脚本 (`tests/e2e_flow.sh`)

```bash
#!/bin/bash
# 完整端到端流程：登录 → 聊天 → 结束 → 管理员查看
# 使用方式: bash tests/e2e_flow.sh

set -e
BASE="http://localhost:8000"

echo "===== 端到端测试 ====="

# Step 1: 微信登录
echo "Step 1: 微信登录..."
OPENID="e2e_test_$(date +%s)"
R=$(curl -s -X POST $BASE/api/auth/wechat-login \
  -H "Content-Type: application/json" \
  -d "{\"openid\": \"${OPENID}\", \"display_name\": \"E2E测试用户\"}")
TOKEN=$(echo "$R" | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")
USER_ID=$(echo "$R" | python3 -c "import json,sys; print(json.load(sys.stdin)['user']['id'])")
echo "  -> user_id=$USER_ID, token=${TOKEN:0:20}..."

# Step 2: 创建会话
echo "Step 2: 创建会话..."
R=$(curl -s -X POST $BASE/api/chat/sessions -H "Authorization: Bearer $TOKEN")
SID=$(echo "$R" | python3 -c "import json,sys; print(json.load(sys.stdin)['id'])")
echo "  -> session_id=$SID"

# Step 3: 发送消息（尽可能多发直到对话结束）
echo "Step 3: 发送消息..."
MSGS=(
  "我最近感到很大的压力"
  "主要是学习上的压力，快要考试了"
  "已经连续两周睡不好了"
  "和同学的关系也有些紧张"
  "我爸妈对我期望很高"
  "有时候会很焦虑"
  "上课也集中不了注意力"
  "感觉做什么都提不起劲"
  "对未来感到很迷茫"
  "不知道该跟谁说这些"
)

for i in $(seq 0 9); do
  MSG="${MSGS[$i]}"
  R=$(curl -s -X POST "$BASE/api/chat/sessions/$SID/message" \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d "{\"content\": \"${MSG}\"}")
  
  ENDED=$(echo "$R" | python3 -c "import json,sys; d=json.load(sys.stdin); print(d.get('session_ended',False))" 2>/dev/null)
  REASON=$(echo "$R" | python3 -c "import json,sys; d=json.load(sys.stdin); print(d.get('ending_reason','N/A'))" 2>/dev/null)
  
  NUM=$((i+1))
  if [ "$ENDED" = "True" ]; then
    echo "  -> 消息 $NUM: 会话结束 (reason=$REASON)"
    break
  fi
  echo "  -> 消息 $NUM: OK"
  sleep 1
done

# Step 4: 验证会话状态
echo "Step 4: 验证会话状态..."
R=$(curl -s "$BASE/api/chat/sessions/$SID/messages" -H "Authorization: Bearer $TOKEN")
USER_COUNT=$(echo "$R" | python3 -c "import json,sys; msgs=json.load(sys.stdin); print(sum(1 for m in msgs if m['role']=='user'))")
AI_COUNT=$(echo "$R" | python3 -c "import json,sys; msgs=json.load(sys.stdin); print(sum(1 for m in msgs if m['role']=='assistant'))")
echo "  -> 用户消息: $USER_COUNT, AI回复: $AI_COUNT"

# Step 5: 管理员登录并查看
echo "Step 5: 管理员登录并查看..."
R=$(curl -s -X POST $BASE/api/auth/admin-login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123456"}')
ADMIN_T=$(echo "$R" | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")

# 仪表盘
R=$(curl -s $BASE/api/admin/dashboard -H "Authorization: Bearer $ADMIN_T")
echo "  -> 仪表盘: $(echo $R | python3 -c "import json,sys; d=json.load(sys.stdin); print(f'visitors={d[\"total_visitors\"]}, sessions={d[\"total_sessions\"]}')" 2>/dev/null)"

# 来访者列表
R=$(curl -s $BASE/api/admin/visitors -H "Authorization: Bearer $ADMIN_T")
echo "  -> 来访者: $(echo $R | python3 -c "import json,sys; d=json.load(sys.stdin); print(len(d))" 2>/dev/null) 人"

echo ""
echo "===== 端到端测试完成 ====="
echo "测试用户: $USER_ID"
echo "会话ID: $SID"
echo "用户消息数: $USER_COUNT"
echo "AI回复数: $AI_COUNT"
```
