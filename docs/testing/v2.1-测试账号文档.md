## 二、测试场景与预期结果

### 场景 1：来访者登录 → 选择咨询师 → 查看可预约时间

**操作步骤：**
1. 打开 http://localhost:3000
2. 点击「微信一键登录」（H5 模式自动 mock）
3. 在首页点击「预约咨询」
4. 查看咨询师列表

**预期结果：**
- 能看到 3 位咨询师，各自显示专业领域和个人简介
- 点击陈思远 → 看到上午时段（09:00-12:00）可选日期
- 点击李华 → 看到下午时段（14:00-17:00）
- 点击王芳 → 看到晚间时段（19:00-21:00）

**测试重点：** 检查咨询师是否按 `is_approved=true` 过滤显示；无可用时间的旧咨询师（test_counselor、new_counselor）已被禁用或不在列表中。

### 场景 2：咨询师登录 → 查看工作台

**操作步骤：**
1. 在首页点击「去登录」→ 输入任一咨询师账号密码
2. 登录后查看工作台

**预期结果：**
- 登录成功，无强制修改密码提示
- 工作台正常显示

### 场景 3：来访者完成预约全流程

**操作步骤：**
1. 来访者登录 → 预约咨询 → 选择张明
2. 选择一个可预约日期 → 选择一个时间段
3. 填写预约原因 → 提交

**预期结果：**
- 如未完成心理测评，会提示先完成 GAD-7 和 PHQ-9
- 完成测评后，可成功提交预约

### 场景 4：咨询师查看和管理预约

**操作步骤：**
1. 咨询师登录 → 工作台 → 来访者预约
2. 查看收到的预约请求

**预期结果：**
- 能看到来访者提交的预约
- 可进行确认/拒绝操作

## 三、边缘情况测试清单

- [ ] **重复创建** — 多次运行种子脚本不会创建重复账号
- [ ] **日期边界** — 可预约日期在今天+2~今天+7之间，过期不可见
- [ ] **周末过滤** — 预约时段自动跳过周六日
- [ ] **时段不重叠** — 同一咨询师在同一天的时段不会重叠
- [ ] **认证过滤** — 未认证咨询师不出现在列表中
- [ ] **账号禁用** — 测试管理账号自动禁用后无法登录
- [ ] **空专业领域** — 旧咨询师没有 specialties 时列表显示无异常
- [ ] **密码统一** — 所有测试咨询师密码均为 123456

## 四、数据清理

如需清除测试数据，执行以下 SQL：

```sql
-- 删除测试咨询师（会级联删除关联的可用时间）
DELETE FROM availabilities WHERE counselor_id IN (
  SELECT id FROM users WHERE username IN ('chensiyuan', 'lihua', 'wangfang')
);
DELETE FROM users WHERE username IN ('chensiyuan', 'lihua', 'wangfang');
```

或者直接删除数据库文件重新初始化：

```bash
rm backend/ai_psy_counselor.db
cd backend && python3 utils/seed_admin.py
```
